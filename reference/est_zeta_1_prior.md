# Estimate Prior Distribution for Symptomatic Shedding Rate (zeta_1)

Compiles a meta-analysis of published *Vibrio cholerae* shedding anchors
(stool concentration x time-averaged daily stool volume) for symptomatic
infections and fits a lognormal prior on the per-person-per-day shedding
rate used in the MOSAIC environmental reservoir update: \$\$W(t+1)
\mathrel{+}= (1 - \theta_j) \cdot \mathrm{Poisson}(\zeta_1 \cdot
I\_{\mathrm{sym}})\$\$ so \\\zeta_1\\ is the expected total *V.
cholerae* cells shed per symptomatic infected person per day during the
shedding phase. The \\\gamma_1\\ parameter handles the shedding-phase
duration; \\\zeta_1\\ is a daily rate, not a total-per-infection.

## Usage

``` r
est_zeta_1_prior(
  PATHS,
  severity_mix = c(severe = 0.2, moderate = 0.4, mild = 0.4)
)
```

## Arguments

- PATHS:

  A list of paths generated by
  [`get_paths()`](https://institutefordiseasemodeling.github.io/MOSAIC-pkg/reference/get_paths.md).
  Must include:

  - **MODEL_INPUT**: directory where parameter CSVs are saved.

  - **DOCS_FIGURES**: directory where the summary figure is saved.

- severity_mix:

  Named numeric vector with entries `severe`, `moderate`, and `mild`
  that sum to 1. Used to compute the severity-weighted pool anchor.
  Defaults to `c(severe = 0.2, moderate = 0.4, mild = 0.4)` (endemic
  mix). The severe fraction (0.2) is informed by Harris 2012's narrative
  that approximately 20 percent of symptomatic cholera cases are severe;
  the moderate/mild split is an expert estimate exposed here as a
  sensitivity argument.

## Value

Invisibly returns a list with elements:

- `data`: the source meta-analysis table.

- `fit`: fitted lognormal parameters (`meanlog`, `sdlog`, `median`,
  `mode`, `mean`, `ci_lower`, `ci_upper`, `n_sources`, `mu_log10`,
  `sd_log10`).

- `param_df`: long-format parameter data frame (as saved to disk).

- `prediction`: density grid on linear and log10-x scales.

- `sensitivity`: leave-one-anchor-out sub-fits (`include_calib`,
  `volume_peak`, `volume_low`).

## Details

The function assembles a literature table of per-person-per-day shedding
anchors constructed as stool concentration (\\c\\, cells per mL) times
time-averaged daily stool volume (\\V\\, mL per day) for defined
severity classes (severe, moderate, mild). A severity-weighted pool is
also constructed using the `severity_mix` argument. A weighted lognormal
is then fit by maximum-likelihood on the log10 scale.

**Assembly equation.** Each literature entry contributes \\\zeta\_{1,i}
= c_i \times V_i\\ with lower and upper bounds taken from the spread of
the concentration and volume anchors for that severity class.
Time-averaged stool volumes are used (`V_sev` = 8, `V_mod` = 4, `V_mild`
= 0.5 L/day) because the LASER Poisson update multiplies the rate by the
current `I_sym` compartment, which already integrates the shedding
course via \\\gamma_1\\. Peak purging rates (~1 L/h, ~14-24 L/day) are
treated as upper-bound sensitivity rows, not the central anchor.

**Severity-weighted pool caveat.** At the default severity mix (0.2 /
0.4 / 0.4) the severe class dominates ~94 percent of the arithmetic
because \\c\_{\mathrm{sev}} = 10^8\\ is 100x larger than
\\c\_{\mathrm{mild}} = 10^6\\. This is acceptable: surveillance observes
severe cases disproportionately, so the effective \\\zeta_1\\ the data
constrains is close to the severe value anyway. The `severity_mix`
argument is exposed so callers can run sensitivity analyses (e.g. the
outbreak mix 0.35 / 0.40 / 0.25).

**Weighted MLE on log10 scale.** Weighted mean and SD on the log10 scale
are converted to lognormal `meanlog` and `sdlog`: \$\$\mu\_{\log\_{10}}
= \frac{\sum w_i \log\_{10}(\zeta\_{1,i})}{\sum w_i}, \quad
\sigma^2\_{\log\_{10}} = \frac{\sum w_i (\log\_{10}(\zeta\_{1,i}) -
\mu\_{\log\_{10}})^2}{\sum w_i}\$\$ then `meanlog = mu_log10 * ln(10)`
and `sdlog = sigma_log10 * ln(10)`.

**Sensitivity returns.** The `$sensitivity` list contains three
alternative fits: `include_calib` (refits WITH the MOSAIC calibration
posterior row at weight 0.25, producing a much wider sdlog because the
Frame-B-calibrated value is 7 OOM below biological-units literature
anchors), `volume_peak` (rescales all severe-class V=8 L/day rows to
V=12 L/day), and `volume_low` (rescales to V=6 L/day).

**Per-day vs per-infection.** `zeta_1` is a per-day shedding rate during
the symptomatic infectious period; \\\gamma_1^{-1}\\ (~10 d) governs the
duration. The cumulative cells shed per symptomatic infection is
therefore approximately `zeta_1 / gamma_1`, and the per-infection
contribution ratio vs asymptomatic is `zeta_ratio * (gamma_2 / gamma_1)`
(~5x the per-day ratio under current gamma priors). When comparing this
prior to literature that quotes cumulative stool output per infection,
apply the duration multiplier.

**Relation to documented priors.** Earlier MOSAIC-docs narrative states
`zeta_1 ~ Uniform(10^4, 10^8)`. That range derives from the Frame-B
calibration rescaling and is superseded by this lognormal
literature-derived prior. The new prior's 95\\ well above the stale
Uniform upper bound.

`claude/zeta_priors_update/zeta_sources/` (14 PDFs + 32 source notes).
See `claude/zeta_priors_update/plan_zeta_priors_implementation.md` for
the full provenance.
